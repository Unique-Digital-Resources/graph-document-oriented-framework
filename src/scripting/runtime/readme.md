# Runtime Compiler (`src/scripting/runtime/`)

Compiles parsed ASTs into framework registrations.

## Architectural Role

The parser only understands syntax; this module understands semantics. It takes the Abstract Syntax Tree (AST) and bridges it to the strict type system of the `core` engine. It maps declarative DSL concepts into concrete Rust structs that the `Graph`, `CommandRegistry`, and `PluginManager` can use to enforce rules at runtime.

## Design Decisions

- **Type Resolution:** The compiler maps DSL types (like `string` or `boolean`) and their default literals (like `= false`) into the engine's polymorphic `PropertyValue` enum and `PropertyKind` classification (Persistent vs. Transient).
- **Relation Delegation:** The compiler does not validate relation rules itself. It simply extracts the `preset_name` (e.g., `CHILDREN`) and passes it along in the `CompiledNodeSchema`. The actual validation against the `RelationPresetRegistry` happens later when the plugin attempts to add an edge in the live `Graph`.
- **Framework Agnostic Output:** The `CompiledNodeSchema` generated by this compiler is a plain data struct. It doesn't contain any closures or engine state, making it safe to pass across thread boundaries or serialize if needed.

## Files

- **`compiler.rs`**: The `DslCompiler` struct. Contains the `compile` function which maps a `Vec<AstNode>` into a `Vec<CompiledNodeSchema>`. It handles the translation of default values, property kinds, and relation target types.
